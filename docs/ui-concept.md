# UI Concept

The presentation view should use the whole screen as a relationship board. Controls should stay compact: timeline selection, center selection, view selection, zoom basics, filters, and edit mode entry.

Visual direction:

- Photo-like entity cards using fictional placeholder initials.
- Colored group regions.
- Labeled arrows with varied color and weight.
- Sticky-note annotations.
- Center entity emphasis.
- Manual placement that feels curated rather than dashboard-like.

Editing can be more utilitarian, especially for forms and property panels. A future decision is whether editing remains a mode on the presentation canvas or becomes a separate route.
